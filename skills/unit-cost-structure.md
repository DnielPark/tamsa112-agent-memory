# 단가 구조 및 입력 규칙

## 📊 단가 체계 계층

```
📊 내역단가 (최종 설계)
  ↑ 여기서 단가를 선택해서 가져옴
  │
  ├─ 📋 일위대가 (복합단가 — 기초단가 조합)
  ├─ 📐 품셈단가 (표준 품셈)
  ├─ 🏪 표준시장단가 (시장 조사)
  ├─ 📄 견적단가 (업체 견적)
  ├─ 💰 물가정보지 (자재 단가)
  └─ 🔧 실정보고 (현장 변경/이슈)
```

## ✅ 올바른 입력 순서 (정상 흐름)

```
1. 물가정보지 — 자재 단가 (시멘트, 레미콘 등)
2. 품셈단가 — 표준 품셈 (인력/장비/재료)
3. 표준시장단가 — 시장 조사 가격
4. 견적단가 — 업체 견적서 기반
5. 일위대가 — 위 기초단가들을 조합
6. 내역단가 — 최종 적용 단가
```

## 🔧 실정보고 단가 (비정상 흐름)

**다음 상황에서 사용:**
- 현장 변경/설계 변경으로 새로운 단가 필요
- 자재 단가 인상 등 기존 단가로 커버 불가
- 기존 단가 체계에 없는 새로운 공종

**특징:**
- 근거는 "현장 실정보고" 자체
- 추후 정산/설계변경의 근거 자료
- 기존 단가와 비교해서 증감 확인용
- 별도 테이블에서 관리 (기존 체계와 섞이지 않음)

## 📝 단가 입력 예시

| 요청 | 입력 위치 |
|------|----------|
| "품셈에 터파기 8m 추가" | 품셈단가 (unit_cost_standard) |
| "실정보고에 자재운반비 추가" | 실정보고 단가 (unit_cost_field_report) |
| "일위대가에 아스콘포장 추가" | 일위대가 (unit_cost_composite) |
| "물가에 시멘트 단가 수정" | 물가정보지 (unit_cost_price_info) |

## ⚠️ 주의사항

1. **기초 단가 없이 상위 단가 생성 금지**
   - 일위대가를 만들려면 먼저 기초 단가(품셈/시장/견적/물가)가 있어야 함
   - 내역단가는 기존 단가에서 참조해서 생성

2. **실정보고 단가는 예외**
   - 기존 단가 체계 무시하고 현장 상황 반영
   - 단, 추후 정산 시 근거 자료로 활용

3. **수정 요청 시 유형 명시**
   - "품셈에..." / "실정보고에..." / "일위대가에..."
   - 유형을 명시하지 않으면 물어볼 것

# Civil Cost Manager 프로젝트

## 기본 정보
- **GitHub**: https://github.com/DnielPark/civil-cost-manager (공개)
- **로컬**: `~/.openclaw/workspace/civil-cost-manager/`
- **기술 스택**: Flask + SQLite3 + HTML/CSS/JS + openpyxl + pandas
- **포트**: 8080 (5000번 AirPlay 충돌 회피)

## 스크립트

```bash
# GitHub → 맥북 (최신 코드 가져오기)
cd ~/.openclaw/workspace/skills
./cost-manager-pull.sh

# 맥북 → GitHub (변경사항 푸시)
cd ~/.openclaw/workspace/skills
./cost-manager-push.sh "커밋 메시지"
```

## 서버 실행/중지

```bash
cd ~/.openclaw/workspace/skills
./cost-manager-app-on.sh    # 서버 시작 (http://localhost:8080)
./cost-manager-app-off.sh   # 서버 중지
```
